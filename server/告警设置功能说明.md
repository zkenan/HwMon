# 告警设置功能说明

## 功能概述

告警设置功能允许管理员自定义需要监控的硬件类型。当客户端上报硬件信息时，系统只会对已启用的硬件类型进行变更检测，并在检测到变更时触发告警。

## 支持的硬件类型

系统支持监控以下7种硬件类型的变更：

1. **CPU** - 监控CPU型号变化
2. **GPU** - 监控显卡型号变化
3. **内存** - 监控内存容量变化
4. **硬盘** - 监控硬盘数量、型号和容量变化
5. **网卡** - 监控网卡数量和描述变化
6. **主板** - 监控主板制造商和型号变化
7. **BIOS** - 监控BIOS制造商和版本变化

## 默认配置

系统默认的告警设置为：
- ✅ CPU监控：启用
- ✅ GPU监控：启用
- ✅ 内存监控：启用
- ✅ 硬盘监控：启用
- ❌ 网卡监控：禁用
- ❌ 主板监控：禁用
- ❌ BIOS监控：禁用

## 使用方法

### 1. 访问告警设置

1. 登录硬件监控系统Web界面
2. 点击顶部工具栏的 **"邮件配置"** 按钮
3. 在弹出的对话框中，切换到 **"告警设置"** 标签页

### 2. 配置监控项

在告警设置页面中：
- 勾选需要监控的硬件类型
- 取消勾选不需要监控的硬件类型
- 点击 **"保存设置"** 按钮保存配置

### 3. 恢复默认设置

如果需要恢复到默认配置，点击 **"恢复默认"** 按钮即可。

## 工作原理

### 硬件变更检测流程

1. **客户端首次上报**
   - 系统自动创建该客户端的硬件基准
   - 记录当前所有硬件信息作为对比基准

2. **客户端后续上报**
   - 系统读取当前的告警设置
   - 仅对比已启用监控的硬件类型
   - 如果检测到变更，创建告警记录
   - 如果启用了邮件通知，发送告警邮件

3. **告警触发条件**
   - 只有当被监控的硬件类型发生变更时才会触发告警
   - 未启用监控的硬件类型即使发生变更也不会触发告警

### 示例场景

**场景1：只监控CPU和内存**
- 配置：启用CPU和内存监控，禁用其他
- 结果：只有CPU或内存变更时会触发告警，GPU、硬盘等变更不会告警

**场景2：全面监控**
- 配置：启用所有硬件类型监控
- 结果：任何硬件变更都会触发告警

**场景3：只监控关键硬件**
- 配置：只启用CPU、GPU、内存、硬盘监控
- 结果：核心硬件变更会告警，网卡、主板、BIOS变更不会告警

## API接口

### 获取告警设置

```
GET /api/alert-settings
```

响应示例：
```json
{
  "status": "success",
  "data": {
    "monitor_cpu": 1,
    "monitor_gpu": 1,
    "monitor_memory": 1,
    "monitor_disk": 1,
    "monitor_network": 0,
    "monitor_motherboard": 0,
    "monitor_bios": 0
  }
}
```

### 更新告警设置

```
PUT /api/alert-settings
Content-Type: application/json

{
  "monitor_cpu": true,
  "monitor_gpu": true,
  "monitor_memory": true,
  "monitor_disk": true,
  "monitor_network": false,
  "monitor_motherboard": false,
  "monitor_bios": false
}
```

响应示例：
```json
{
  "status": "success",
  "message": "告警设置已更新"
}
```

## 数据库结构

告警设置存储在 `alert_settings` 表中：

```sql
CREATE TABLE alert_settings (
    id INTEGER PRIMARY KEY CHECK (id = 1),
    monitor_cpu INTEGER DEFAULT 1,
    monitor_gpu INTEGER DEFAULT 1,
    monitor_memory INTEGER DEFAULT 1,
    monitor_disk INTEGER DEFAULT 1,
    monitor_network INTEGER DEFAULT 0,
    monitor_motherboard INTEGER DEFAULT 0,
    monitor_bios INTEGER DEFAULT 0,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

## 注意事项

1. **全局设置**：告警设置是全局的，对所有客户端生效
2. **即时生效**：修改设置后立即生效，无需重启服务
3. **基准数据**：修改设置不会影响已有的硬件基准数据
4. **历史告警**：修改设置不会影响已产生的告警记录
5. **建议配置**：根据实际需求选择监控项，避免过多不必要的告警

## 最佳实践

1. **生产环境**：建议启用CPU、GPU、内存、硬盘监控，这些是关键硬件
2. **测试环境**：可以根据需要灵活配置
3. **特殊需求**：如果需要监控硬件更换（如网卡升级），可以启用相应监控
4. **减少噪音**：对于经常变动但不重要的硬件，可以禁用监控以减少告警噪音

## 故障排查

### 问题1：修改设置后不生效

**检查步骤：**
1. 确认已成功保存设置（查看提示信息）
2. 刷新页面重新加载设置
3. 检查浏览器控制台是否有错误

### 问题2：仍然收到不需要的告警

**解决方法：**
1. 检查告警设置是否正确配置
2. 确认客户端使用的是最新的基准数据
3. 如有必要，重置客户端基准

### 问题3：API返回错误

**常见原因：**
1. 服务器未启动
2. 数据库文件损坏
3. 权限问题

**解决方法：**
1. 检查服务器日志
2. 重启服务器
3. 检查数据库文件权限
