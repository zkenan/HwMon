为硬件监控系统添加了登录验证功能，保护Web管理界面和API接口，防止未授权访问。

## 🔧 技术实现

### 后端实现

#### 1. Session配置
```python
app.secret_key = 'hardware_monitor_secret_key_2026'
LOGIN_CONFIG = {
    'username': 'xapi',
    'password': 'Ai78965'
}
```

#### 2. 登录验证装饰器
```python
def login_required(f):
    @functools.wraps(f)
    def decorated_function(*args, **kwargs):
        if 'logged_in' not in session:
            if request.path.startswith('/api/'):
                return jsonify({'error': '未登录', 'need_login': True}), 401
            return redirect(url_for('login'))
        return f(*args, **kwargs)
    return decorated_function
```

#### 3. 登录API
```python
@app.route('/api/login', methods=['POST'])
def api_login():
    data = request.json
    username = data.get('username', '')
    password = data.get('password', '')
    
    if username == LOGIN_CONFIG['username'] and password == LOGIN_CONFIG['password']:
        session['logged_in'] = True
        session['username'] = username
        return jsonify({'status': 'success', 'message': '登录成功'})
    else:
        return jsonify({'status': 'error', 'message': '用户名或密码错误'}), 401
```

#### 4. 受保护的API
所有管理API都添加了 `@login_required` 装饰器：
- `/api/clients/*` - 客户端管理
- `/api/groups/*` - 分组管理
- `/api/collect/*` - 采集控制
- `/api/export/*` - 数据导出
- `/api/alerts/*` - 告警管理
- `/api/email-config/*` - 邮件配置
- `/api/alert-settings/*` - 告警设置
- 等等...

**例外**:
- `/api/report` - 客户端上报（无需登录）
- `/api/login` - 登录接口（无需登录）
- `/api/logout` - 登出接口（无需登录）
- `/api/check-login` - 检查登录状态（无需登录）

## 🔒 安全性

### 当前安全措施
1. ✅ Session加密（secret_key）
2. ✅ 密码硬编码在服务器端
3. ✅ HTTPS建议（生产环境）
4. ✅ API访问控制
5. ✅ 未登录自动重定向

### 安全建议

#### 生产环境部署
1. **修改默认密码**
   ```python
   LOGIN_CONFIG = {
       'username': 'your_username',
       'password': 'your_strong_password'
   }
   ```

2. **启用HTTPS**
   - 使用SSL证书
   - 加密传输数据
   - 防止中间人攻击

3. **修改secret_key**
   ```python
   import os
   app.secret_key = os.urandom(24)
   ```

4. **添加IP白名单**（可选）
   ```python
   @app.before_request
   def check_ip():
       allowed_ips = ['127.0.0.1', '192.168.1.100']
       if request.remote_addr not in allowed_ips:
           return jsonify({'error': 'IP不允许'}), 403
   ```

5. **添加登录尝试限制**
   - 防止暴力破解
   - 记录登录日志
   - 失败多次后锁定

6. **使用数据库存储用户**
   - 支持多用户
   - 密码加密存储
   - 角色权限管理

## ⚠️ 注意事项

1. **首次使用**
   - 确保使用了正确的用户名和密码
   - 区分大小写

2. **会话保持**
   - 不要清除浏览器Cookie
   - 关闭浏览器后会话失效

3. **多标签页**
   - 同一浏览器多个标签页共享会话
   - 一个标签页登出，其他也会失效

4. **客户端上报**
   - 客户端上报不需要登录
   - 这是为了让客户端能自动上报数据

5. **密码安全**
   - 不要在公共场合输入密码
   - 定期更换密码（生产环境）

## 🔧 故障排查

### 问题1: 无法登录

**可能原因**:
- 用户名或密码错误
- Session配置问题
- 浏览器禁用了Cookie

**解决方法**:
1. 确认用户名是 `xapi`（小写）
2. 确认密码是 `Ai78965`（注意大小写）
3. 检查浏览器是否允许Cookie
4. 清除浏览器缓存后重试

### 问题2: 登录后仍然提示未登录

**可能原因**:
- Session未正确保存
- secret_key配置问题

**解决方法**:
1. 检查服务器日志
2. 确认 `app.secret_key` 已设置
3. 重启服务器
4. 清除浏览器Cookie后重新登录

### 问题3: 客户端无法上报数据

**可能原因**:
- 误将 `/api/report` 也加了登录验证

**解决方法**:
- 确认 `/api/report` 没有 `@login_required` 装饰器
- 客户端上报API应该是公开的

### 问题4: 退出登录后仍能访问

**可能原因**:
- 浏览器缓存
- Session未清除

**解决方法**:
1. 清除浏览器缓存
2. 检查 `api_logout` 函数是否正确清除session
3. 刷新页面
