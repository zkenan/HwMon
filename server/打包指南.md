## 🔧 常见问题

### Q1: 打包失败怎么办？

**常见原因**:
1. PyInstaller未安装
2. 缺少依赖包
3. templates文件夹不存在

**解决方法**:
```bash
# 重新安装依赖
pip install -r requirements.txt

# 清理缓存后重新打包
rmdir /s /q build dist
python build_exe.py
```

### Q2: exe文件太大？

**正常现象**: 
- 单文件模式包含完整Python环境
- 大小约60-80MB是正常的
- 目标机器无需安装Python

**减小体积**:
- 使用 `--onedir` 模式（多文件）
- 排除不必要的模块
- 使用UPX压缩（需安装UPX）

### Q3: 启动后无法访问？

**检查清单**:
1. ✅ 端口5000是否被占用
2. ✅ 防火墙是否开放端口
3. ✅ 是否正确输入登录凭据
4. ✅ 浏览器地址是否正确

**解决方法**:
```bash
# 检查端口占用
netstat -ano | findstr :5000

# 开放防火墙端口（管理员权限）
netsh advfirewall firewall add rule name="HardwareMonitor" dir=in action=allow protocol=TCP localport=5000
```

### Q4: 如何后台运行？

**方法1**: 使用提供的脚本
```
双击 "后台运行.bat"
```

**方法2**: 使用Windows任务计划程序
1. 打开任务计划程序
2. 创建基本任务
3. 触发器：系统启动时
4. 操作：启动程序 `HardwareMonitorServer.exe`

**方法3**: 注册为Windows服务（需要额外工具）
- 使用NSSM (Non-Sucking Service Manager)
- 或使用pywin32创建服务

### Q5: 如何更新版本？

**步骤**:
1. 停止当前运行的服务端
2. 备份数据库文件 `hardware_monitor.db`
3. 替换新的 `HardwareMonitorServer.exe`
4. 恢复数据库文件
5. 重新启动服务端

## 📊 性能优化

### 并发配置

编辑 `app.py`:
```python
COLLECT_CONFIG = {
    'max_workers': 50,        # 最大并发数
    'timeout': 15,            # 超时时间(秒)
    'retry_times': 0,         # 重试次数
}
```

根据服务器性能调整：
- **低配服务器**: max_workers=20
- **中配服务器**: max_workers=50
- **高配服务器**: max_workers=100

### Waitress配置

编辑 `app.py` 最后一行：
```python
serve(app, host='0.0.0.0', port=5000, threads=20, connection_limit=2000)
```

参数说明：
- `threads`: 工作线程数（建议CPU核心数×2）
- `connection_limit`: 最大连接数

## 🔒 安全建议

### 生产环境部署

1. **修改默认密码**
   ```python
   LOGIN_CONFIG = {
       'username': 'admin',
       'password': 'YourStrongPassword123!'
   }
   ```

2. **启用HTTPS**（需要反向代理）
   - 使用Nginx作为反向代理
   - 配置SSL证书
   - 将HTTP重定向到HTTPS

3. **限制访问IP**
   在 `app.py` 添加中间件：
   ```python
   @app.before_request
   def check_ip():
       allowed_ips = ['127.0.0.1', '192.168.1.0/24']
       if request.remote_addr not in allowed_ips:
           return jsonify({'error': 'IP不允许'}), 403
   ```

4. **定期备份数据库**
   ```bash
   # Windows任务计划程序
   # 每天凌晨2点备份
   xcopy hardware_monitor.db backup\hardware_monitor_%date:~0,4%%date:~5,2%%date:~8,2%.db /Y
   ```
