
### 问题2: API返回404错误

**可能原因**:
- 后端代码未更新
- 服务未重启

**解决方法**:
```bash
# 确认app.py已更新
# 检查是否包含 get_alert_settings 和 update_alert_settings 函数

# 重启服务
python app.py
```

### 问题3: 数据库错误

**可能原因**:
- 数据库表创建失败
- 权限问题

**解决方法**:
```python
# 手动创建表（在Python中执行）
import sqlite3

conn = sqlite3.connect('hardware_monitor.db')
cursor = conn.cursor()

cursor.execute('''
    CREATE TABLE IF NOT EXISTS alert_settings (
        id INTEGER PRIMARY KEY CHECK (id = 1),
        monitor_cpu INTEGER DEFAULT 1,
        monitor_gpu INTEGER DEFAULT 1,
        monitor_memory INTEGER DEFAULT 1,
        monitor_disk INTEGER DEFAULT 1,
        monitor_network INTEGER DEFAULT 0,
        monitor_motherboard INTEGER DEFAULT 0,
        monitor_bios INTEGER DEFAULT 0,
        updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
''')

cursor.execute('INSERT OR IGNORE INTO alert_settings (id) VALUES (1)')
conn.commit()
conn.close()

print("表创建成功")
```

### 问题4: 保存设置失败

**可能原因**:
- 数据库写入权限不足
- 数据库文件被锁定

**解决方法**:
```bash
# Windows: 检查文件权限
# 右键 hardware_monitor.db -> 属性 -> 安全
# 确保当前用户有写入权限

# Linux/Mac: 检查文件权限
ls -l hardware_monitor.db
chmod 644 hardware_monitor.db
```
