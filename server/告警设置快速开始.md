# 告警设置功能 - 快速开始

## 🎯 功能简介

告警设置功能让您能够灵活控制哪些硬件变更会触发告警。您可以：
- ✅ 只监控CPU变更
- ✅ 只监控内存变更  
- ✅ 监控所有硬件
- ✅ 自定义任意组合

## 🚀 快速上手

### 步骤1：打开系统配置

在Web界面顶部工具栏，点击 **"邮件配置"** 按钮（现在改名为"系统配置"）

### 步骤2：切换到告警设置

在弹出的对话框中，您会看到两个标签页：
- **邮件配置** - 配置SMTP邮件发送
- **告警设置** - 配置硬件监控规则 ⬅️ 点击这里

### 步骤3：选择监控项

您会看到7个硬件类型的选项卡片：

```
┌─────────────┐  ┌─────────────┐
│ ☑ CPU       │  │ ☑ GPU       │
│ 监控CPU型号  │  │ 监控显卡型号 │
└─────────────┘  └─────────────┘

┌─────────────┐  ┌─────────────┐
│ ☑ 内存      │  │ ☑ 硬盘      │
│ 监控容量变化 │  │ 监控数量型号 │
└─────────────┘  └─────────────┘

┌─────────────┐  ┌─────────────┐
│ ☐ 网卡      │  │ ☐ 主板      │
│ 监控网卡变化 │  │ 监控主板信息 │
└─────────────┘  └─────────────┘

┌─────────────┐
│ ☐ BIOS      │
│ 监控BIOS版本 │
└─────────────┘
```

**操作方式：**
- 点击卡片或复选框来启用/禁用监控
- 勾选 = 该硬件变更时会告警
- 不勾选 = 该硬件变更时不会告警

### 步骤4：保存设置

点击底部的 **"保存设置"** 按钮

✅ 完成！设置立即生效

**优点：** 任何硬件变更都能及时发现

## 🔧 高级操作

### 恢复默认设置

如果您修改了设置但想恢复到初始状态：

1. 在告警设置页面
2. 点击 **"恢复默认"** 按钮
3. 会自动设置为：CPU、GPU、内存、硬盘启用，其他禁用

### 通过API配置

您也可以使用API进行配置：

```bash
# 获取当前设置
curl http://localhost:5000/api/alert-settings

# 更新设置（只监控CPU）
curl -X PUT http://localhost:5000/api/alert-settings \
  -H "Content-Type: application/json" \
  -d '{
    "monitor_cpu": true,
    "monitor_gpu": false,
    "monitor_memory": false,
    "monitor_disk": false,
    "monitor_network": false,
    "monitor_motherboard": false,
    "monitor_bios": false
  }'
```

## ❓ 常见问题

### Q1: 修改设置后，之前的基准数据会受影响吗？

**A:** 不会。告警设置只影响变更检测的逻辑，不会影响已保存的硬件基准数据。

### Q2: 如果禁用了某项监控，之前的告警记录会删除吗？

**A:** 不会。历史告警记录会保留，只是新的变更不会再触发该类告警。

### Q3: 不同客户端可以有不同的监控设置吗？

**A:** 目前告警设置是全局的，对所有客户端统一生效。如需针对不同客户端设置，可以联系开发团队定制。

### Q4: 设置保存后需要重启服务吗？

**A:** 不需要。设置保存后立即生效，新上报的客户端数据会使用新配置进行检测。

### Q5: 如何知道当前哪些硬件在监控中？

**A:** 打开告警设置页面，勾选的项目即为正在监控的硬件类型。
